# 一.**概要**

本資料は、八嶋諸邦における公文書体系「式」の定義・構造・運用規則を定めるものである。「式」とは、国家が現時点において有効と定めた規範の総体であり、祭祀・行政その他の領域ごとに種別を分けて編纂される。

---
# 二.**式の定義**
## 二.一 式とは何か

式とは、八嶋諸邦における規範文書の正式名称である。各領域に関する規定・手続き・禁止事項等を条文形式で記述したものであり、国家が有効と定めた唯一の現行規範として機能する。

式は以下の性質を持つ。

- **現行性**：常に最新の改訂版のみが有効な規範として扱われる。
- **規範性**：条文は規定・指示の形式で記述され、推測・断定・説明を含まない。
- **完結性**：条文はそれ単独で解釈可能なものとし、他文書への依存を原則として設けない。

## 二.二 式の文体

式の条文は中古日本語（書き下し文）の文体を用いて記述する。現代語・口語・説明的文章は用いない。条文は規定・指示の形式とし、理由・背景・因果の記述を含めない。

---
# 三.**式の種別**
## 三.一 現行の種別一覧

式は以下の種別に分類される。種別は必要に応じて追加する。

|種別|対象領域|
|---|---|
|儀式|祭祀・御神体・神域に関する事項|
|官務式|中央・地方行政の手続きに関する事項|

## 三.二 種別の追加

新たな種別の追加は、既存の種別では対応できない領域が生じた場合に限り行う。追加にあたっては本体系設計文書を改訂する。

---

# 四.**文書の三層構造**

## 四.一 構造の概要

式の文書体系は以下の三層によって構成される。各層は明確に区別され、相互に混在させてはならない。

|層|名称|役割|
|---|---|---|
|第一層|現行|現時点で有効な唯一の規範本体|
|第二層|原案|改訂前の現行を保存した履歴庫|
|第三層|解釈|改訂理由を記録した設計ログ|

## 四.二 現行

現行は、現時点において有効な規範の本体である。各式につき常に一つのみ存在し、改訂のつど上書きされる。読者が参照すべき文書は現行のみである。 新規作成したファイルはここに置く。改訂された際には原案フォルダへ移動する。

## 四.三 原案

原案は、改訂前の現行をそのまま保存したものである。改訂の都度、旧現行を原案フォルダへ移動して蓄積する。原案は履歴の確認・比較のためにのみ存在し、規範としての効力を持たない。原案の内容を事後的に変更してはならない。

## 四.四 解釈

解釈は、現行の改訂理由を記録する設計ログである。現行ファイルの末尾に独立した章として設ける。解釈には以下の二種を記録する。

- **メタ改訂**：他の設定との整合性調整を目的とした改訂
- **欠陥発覚**：運用上の問題・矛盾の発覚による改訂

解釈は完全なメタ文書として記述し、世界観的表現・神話的文体・設定内容を含めてはならない。 新規作成時および未改訂の場合、解釈の各項目には「なし」と記載する。

# 五.**式の本文構造**

## 五.一 構造単位

式の本文は以下の三単位によって構成される。

|単位|説明|
|---|---|
|編|式全体の最上位区分|
|章|編の下位区分。編ごとに第一章から番号を振り直す|
|条|章の下位区分。章ごとに第一条から番号を振り直す|

編・章・条はすべて必須とする。条のみによる構成は認めない。

## 五.二 番号の振り方

- 編の番号は式全体で通し番号とする（第一編・第二編…）。
- 章の番号は編ごとに振り直す（各編の第一章から始まる）。
- 条の番号は章ごとに振り直す（各章の第一条から始まる）。

## 五.三 記述形式の例

```
# 第一編　○○
## 第一章　○○
### 第一条
（条文）
### 第二条
（条文）
## 第二章　○○
### 第一条
（条文）
# 第二編　○○
## 第一章　○○
### 第一条
（条文）
```

# 六.**改訂の手続き**

## 六.一 改訂プロセス

式の改訂は以下の順序に従って行う。

1. 現行の改訂が必要と判断される。
2. 現行フォルダにある旧版を原案フォルダへ移動する。
3. 改訂版を新規に作成し、現行フォルダへ配置する。
4. 必要に応じて解釈に改訂理由を記録する。

## 六.二 読む対象・読まない対象

|対象|取り扱い|
|---|---|
|現行|常に参照する|
|解釈|必要に応じて参照する|
|原案|原則として参照しない|

# 七.**YAMLフロントマター**

## 七.一 記載項目

式の現行ファイルには以下のYAMLフロントマターを記載する。

```yaml
---
制定日: （YYYY-MM-DD を漢数字で記述）
改定日: （YYYY-MM-DD を漢数字で記述。未改定の場合は / とする）
---
```

## 七.二 日付の表記形式

日付はYYYY-MM-DD形式を漢数字で記述する。

例：二〇二六-〇五-二五

# 八.**ファイル管理規則**

## 八.一 ディレクトリ構成

各式は以下のディレクトリ構成に従って管理する。

```
式/
├── 儀式/
│   ├── 現行/
│   │   └── 第n巻「〇〇」.md
│   └── 原案/
│       └── 第n巻「〇〇」.md
└── 官務式/
    ├── 現行/
    │   └── 第n巻「〇〇」.md
    └── 原案/
        └── 第n巻「〇〇」.md
```

## 八.二 ファイル命名規則

巻番号は式種別ごとに第一巻から始まる。現行・原案ともに同一のファイル名を使用し、格納フォルダによって区別する。

|種別|命名形式|例|
|---|---|---|
|現行|第n巻「巻名」|第一巻「神事心得」|
|原案|第n巻「巻名」|第一巻「神事心得」|

解釈は現行ファイルの末尾章として記述するため、独立したファイルを持たない。
